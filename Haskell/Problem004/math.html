Proof that, in the square multiplication table, the diagonal of squares describes a "ridge line" (imagine a roof),and as you "move off" the ridge, you go "downhill" (toward smaller numbers).

So, for example, 900^2 = 810000, but moving off the ridge (perpendicularly), you get smaller numbers, so901*899 =...

(%i2) 901*899;

...a smaller number. Generalized proof follows. Pick any (x,y) pair of integers and move any distance e along thesouthwest-northeast axis (where (1,1) is the top-left corner of the table).

(%i3) x*y-(x-e)*(y+e);

(%i4) expand(x*y-(x-e)*(y+e));

Or...

(%i5) factorsum(x*y-(x-e)*(y+e));

If you think of positions BELOW the main diagonal and moving toward the southwest, so that e is a positive number andy is always greater than x, then you always get positive numbers (i.e., x decreses and y increases).  The rest of this writeup assumes we're belowthat main diagonal.  (The logic is the same for above the diagonal, obviously, although signs change.)

And now, the d3 function I mentioned in the code comments:

(%i6) d3(n,m):=(836+n)*(836+m)-836^2;

(%i7) ratsimp(d3(n,m));

Find an expression that gives us m for a given n such that the delta will still be zero

(%i8) solve(d3(n,m)=0,m);

Make the solution a function, "m for n".

(%i9) m4(n):= 836*n/(n+836);

Test. Back up one position on the x axis. (-1)

(%i10) m4(-1);

Need to shift y more than 1 (+1).  Verify:

(%i11) d3(-1,1);

(%i12) d3(-1,2);

What's the biggest y-offset from 836 we can tolerate? 999-836.

(%i13) 999-836;

What x-offset will require that large a y-offset?

(%i14) solve(m4(n)=-163,n),numer;

Verify. Back x off by -136 and increase y the maximum (so... is (836-136)*(836+163) > 836^2 ?):

(%i15) d3(-136,163);

(%i16) d3(-137,163);

Here's another way to do it, that doesn't require an alternative function like m4:

(%i17) solve(d3(n,163)=0,n),numer;

So, how did I know the relationship between n and m isn't linear? For starters, the function doesn't have a linearform, but also, if you plot it, it doesn't look like a straight line.

(%i22) plot2d(m4(n),[n,-136,0]);

The real proof is in the derivative.  Straight line ==> constant derivative.

(%i19) diff(m4(n),n);

(%i20) ratsimp(diff(m4(n),n),n);

That ain't a constant, so the m4 function isn't linear.

(%i23) plot2d(diff(m4(n),n),[n,-136,0]);

So, never mind trying to be too fancy.  Just take all pairs in the triangle starting at (700,700) (which is 836 - 136)and find the biggest palindrome.  Not super-elegant, but not TOO wasteful.
